## 1. Purpose of this report

This report describes the user-facing minimal release package generated by:

```text
tools/build_minimal_release_package.py
```

The package name is:

```text
sed_reference_release_minimal
```

The minimal package is derived from the full S8 release package `sed_reference_release`. It is designed as a lightweight, model-ready, station-by-time matrix package for users who primarily need daily, monthly, or annual river suspended sediment reference observations without opening the full provenance-rich release directory.

Compared with the full S8 release package, the minimal package intentionally removes heavy or specialist products such as the master long-table NetCDF archive, climatology product, satellite validation-only product, GIS sidecars, overlap-candidate table, and other non-essential sidecars. It keeps only the core matrix NetCDF files and compact catalog tables required for common user workflows.

The resulting minimal release product is intended for:

- quick model-observation comparison;
- station-level time-series extraction;
- direct use of daily, monthly, and annual `Q`, `SSC`, and `SSL` matrices;
- basic station screening by location, temporal coverage, and available resolution;
- lightweight source-station provenance tracing through `selected_source_station_uid` and `source_station_catalog.csv`;
- public distribution when the full S8 release package is too large or too complex for first-time users.

## 3. Relationship to the full S8 release package

The minimal package is built from the full S8 release directory. The full release is expected to contain at least the following input files:

```text
sed_reference_timeseries_daily.nc
sed_reference_timeseries_monthly.nc
sed_reference_timeseries_annual.nc
station_catalog.csv
source_station_catalog.csv
source_dataset_catalog.csv
```

Only these files are required to build the minimal package. The three matrix NetCDF files are copied in a slimmed form, with only selected variables retained. The three catalog CSV files are filtered and reduced to compact user-facing columns.

The following products are intentionally excluded from `sed_reference_release_minimal`:

```text
sed_reference_master.nc
sed_reference_climatology.nc
sed_reference_satellite.nc
satellite_catalog.csv
sed_reference_overlap_candidates.csv.gz
*.gpkg
*.parquet
```

This design separates a simple user-facing matrix package from advanced provenance, climatology, satellite-validation, GIS, and overlap-diagnostic products.

## 4. Product-level design

### 4.1 Main design concept

The minimal package is organized around three linked layers:

```text
Cluster / virtual station layer
        │
        ├── Station-time matrix layer
        │
        └── Selected source-station provenance layer
```

Each row in a matrix NetCDF file represents a merged cluster or virtual reference station. Each column represents a time step for the corresponding temporal resolution. Each populated cell stores selected values for discharge and suspended sediment variables, together with quality flags and a selected source-station identifier.

### 4.2 Core join keys

| Key | Meaning | Main use |
|---|---|---|
| `cluster_uid` | Stable ID for a merged cluster or virtual station, usually formatted as `SED######`. | Primary station join key across NetCDF matrices and `station_catalog.csv`. |
| `cluster_id` | Numeric cluster identifier from the upstream merge workflow. | Compatibility field and internal numeric ID. |
| `resolution` | Temporal resolution of the product or catalog row. | Distinguishes daily, monthly, and annual records. |
| `time` | NetCDF time coordinate. | Time-series alignment and model-observation comparison. |
| `selected_source_station_uid` | Source station selected for a station-time matrix cell. | Cell-level provenance key for tracing a value back to `source_station_catalog.csv`. |
| `source_station_uid` | Stable ID for an original source station, usually formatted as `SRC######`. | Source-station provenance lookup key. |
| `source_name` | Short source dataset name. | Source-dataset summary and citation workflow. |

### 4.3 Temporal resolutions

The minimal package contains only the three main station-reference temporal resolutions:

| Resolution | Product file | Primary use |
|---|---|---|
| Daily | `sed_reference_timeseries_daily.nc` | Daily model validation and event-scale time-series extraction. |
| Monthly | `sed_reference_timeseries_monthly.nc` | Monthly model validation, seasonal analysis, and lower-noise comparison. |
| Annual | `sed_reference_timeseries_annual.nc` | Annual sediment flux evaluation and long-term comparison. |

Climatology and satellite validation-only products are not included in the minimal package. They are handled as separate post-release packages when generated by the same script.

## 5. Core matrix NetCDF products

## 5.1 Shared matrix-product design

The three minimal NetCDF products share the same matrix logic:

```text
Rows    = stations / merged clusters
Columns = time steps for one temporal resolution
Cells   = selected observation values and flags
```

The minimal matrix products keep only variables needed for direct use by most users. The retained variables are:

```text
lat
lon
cluster_id
cluster_uid
time
Q
SSC
SSL
Q_flag
SSC_flag
SSL_flag
n_valid_time_steps
selected_source_station_uid
basin_area
station_name
river_name
```

Among these, the required variables are:

```text
cluster_uid
time
Q
SSC
SSL
Q_flag
SSC_flag
SSL_flag
n_valid_time_steps
```

Optional variables are retained when present in the full S8 matrix files. If an optional variable is missing from a source matrix file, the build script warns about it but can still continue.

### 5.1.1 Main scientific variables

| Variable | Meaning | Typical unit |
|---|---|---|
| `Q` | River discharge | `m3 s-1` |
| `SSC` | Suspended sediment concentration | `mg L-1` |
| `SSL` | Suspended sediment load | `ton day-1` |

### 5.1.2 Quality flags

The package stores variable-level quality flags separately:

```text
Q_flag
SSC_flag
SSL_flag
```

The exact flag meanings are preserved from the source matrix files through variable attributes and the global attribute `qc_flag_meanings`. In the full S8 release convention, the standardized flag values are commonly interpreted as:

| Flag | Meaning |
|---|---|
| `0` | Good |
| `1` | Estimated |
| `2` | Suspect |
| `3` | Bad |
| `9` | Missing |

Users should apply quality-flag filtering according to their analysis purpose. For strict validation, a common choice is to use only values with flag `0`, or values with flags `0` and `1` when estimated values are acceptable.

### 5.1.3 Cell-level provenance

The most important provenance variable retained in the minimal matrix products is:

```text
selected_source_station_uid[n_stations, time]
```

This variable identifies the source station selected for each station-time cell. Users can join it to:

```text
source_station_catalog.csv
```

to recover the original source station, source dataset, native station ID, source station name, river name, original coordinates, and temporal coverage.

Important limitation: the minimal package retains only the selected source-station identifier. It does not include the full `sed_reference_master.nc` record table or the overlap-candidate table. Therefore, it is suitable for basic provenance tracing but not for full overlap diagnostics or selected-vs-unselected candidate comparison.

### 5.1.4 Global attributes retained in matrix files

Each minimal matrix NetCDF file keeps a compact set of global attributes:

```text
title
product_role
release_version
date_created
date_modified
Conventions
summary
variables_provided
qc_flag_meanings
time_coverage_start
time_coverage_end
geospatial_lat_min
geospatial_lat_max
geospatial_lon_min
geospatial_lon_max
```

These attributes help users identify the product role, release version, creation time, variables provided, quality-flag meanings, temporal coverage, and spatial coverage.

### 5.1.5 Compression

The script applies NetCDF compression to the main numeric matrix variables and their flags:

```text
Q
SSC
SSL
Q_flag
SSC_flag
SSL_flag
```

The default requested compression level is `4`. This keeps the package smaller while retaining the matrix layout needed for efficient array-based access.

## 8. What the minimal package does not include

The minimal package is intentionally not a complete provenance archive. Users should switch to the full S8 release package when they need any of the following:

| Need | Required full-release product | Reason |
|---|---|---|
| Record-level long-table archive | `sed_reference_master.nc` | The minimal package stores station-time matrices, not the complete selected-record table. |
| Selected and non-selected overlap candidates | `sed_reference_overlap_candidates.csv.gz` | The minimal package does not retain unselected candidate records. |
| GIS station/basin mapping | `sed_reference_cluster_points.gpkg`, `sed_reference_source_stations.gpkg`, `sed_reference_cluster_basins.gpkg` | GPKG sidecars are excluded from the minimal package. |
| Climatology data | `sed_reference_climatology.nc` | Climatology is separated into an optional standalone package. |
| Satellite validation-only data | `sed_reference_satellite.nc`, `satellite_catalog.csv` | Satellite data are separated into an optional standalone package. |
| Source-level matrix lookup variables | full matrix files in `sed_reference_release` | Variables such as `source_name`, `selected_source_index`, and `is_overlap` are forbidden in the minimal matrix files. |

This separation keeps the minimal package easier to distribute and easier for users to understand, while preserving a path back to the full release package for advanced provenance analysis.

## 9. Recommended user workflows

## 9.1 Model-observation comparison workflow

Recommended workflow:

1. Identify the temporal resolution of the model output.

```text
daily model output   -> sed_reference_timeseries_daily.nc
monthly model output -> sed_reference_timeseries_monthly.nc
annual model output  -> sed_reference_timeseries_annual.nc
```

2. Read `station_catalog.csv`.
3. Filter stations by `resolution`, region, coordinates, time span, and `n_valid_time_steps`.
4. Use `cluster_uid` to locate the corresponding row in the matrix NetCDF file.
5. Extract observed `Q`, `SSC`, or `SSL`.
6. Extract the corresponding quality flag: `Q_flag`, `SSC_flag`, or `SSL_flag`.
7. Align the model and observation time axes.
8. Apply quality-flag filtering.
9. Compute model performance metrics such as bias, RMSE, correlation, NSE, or KGE.
10. Use `selected_source_station_uid` to document source-station provenance.

---

## 9.2 Station search and time-series extraction workflow

Recommended workflow:

1. Open `station_catalog.csv`.
2. Filter to one resolution: `daily`, `monthly`, or `annual`.
3. Select a station by one of the following methods:

```text
cluster_uid
nearest latitude/longitude
country
station_name
river_name
time_start / time_end
n_valid_time_steps
```

4. Open the corresponding matrix file.
5. Match the selected `cluster_uid` to the NetCDF `cluster_uid` variable.
6. Extract the desired variable and its flag.
7. Remove missing values and optionally retain only high-quality records.
8. Save the extracted series as a CSV for downstream analysis.

The repository also provides a minimal example script:

```text
tools/example_reference_workflow_minimal.py
```

A typical command is:

```bash
python3 tools/example_reference_workflow_minimal.py \
  --release-dir output/sed_reference_release_minimal \
  --resolution monthly \
  --lat 30.5 \
  --lon 114.3 \
  --variable SSC \
  --out-csv /tmp/minimal_reference_series.csv
```

This workflow loads the station catalog, finds the nearest station for a chosen resolution, extracts a non-missing reference time series, and optionally writes the result to CSV.

---

## 9.3 Basic provenance tracing workflow

Recommended workflow for tracing a matrix value back to its selected source station:

1. Open one of the matrix files:

```text
sed_reference_timeseries_daily.nc
sed_reference_timeseries_monthly.nc
sed_reference_timeseries_annual.nc
```

2. Select a station-time cell.
3. Read:

```text
cluster_uid
time
selected_source_station_uid
```

4. Open:

```text
source_station_catalog.csv
```

5. Join:

```text
selected_source_station_uid == source_station_uid
```

6. Retrieve source-station metadata:

```text
source_name
source_station_native_id
source_station_id
source_station_name
source_station_river_name
source_station_lat
source_station_lon
time_start
time_end
```

7. Use `source_name` to join to `source_dataset_catalog.csv` for dataset-level metadata, citation information, institution, and source URL.

Important limitation: this workflow identifies the selected source station only. It does not recover unselected overlapping candidates. For overlap diagnostics, use the full S8 release package.

## 10. Building the minimal package

The default command is:

```bash
python3 tools/build_minimal_release_package.py --force
```

By default, the script reads from:

```text
scripts_basin_test/output/sed_reference_release/
```

and writes the minimal package to:

```text
scripts_basin_test/output/sed_reference_release_minimal/
```

The same script can also write optional standalone climatology and satellite packages:

```text
scripts_basin_test/output/sed_reference_release_climatology/
```

## 11. Important release design rules

## 11.1 Minimal means matrix-first

The minimal package is centered on station-by-time matrix products. It is not intended to replace the full S8 release for advanced provenance, overlap diagnostics, GIS workflows, or climatology/satellite analysis.

---

## 11.2 Only daily, monthly, and annual mainline records are included

The package only includes:

```text
daily
monthly
annual
```

Catalog rows are filtered to these three resolutions.

---

## 11.3 Matrix variables are deliberately reduced

The matrix NetCDF files keep only a user-facing subset of variables. The following variables are explicitly forbidden in minimal matrices:

```text
source_name
selected_source_index
is_overlap
```

This prevents the minimal package from carrying heavier full-release source lookup structures or overlap flags that require the full provenance sidecars for correct interpretation.

---

## 11.4 Provenance is compact, not complete

The minimal package supports selected source-station tracing through:

```text
selected_source_station_uid
source_station_catalog.csv
source_dataset_catalog.csv
```

It does not support full selected-vs-unselected candidate comparison. Users requiring complete provenance should use the full S8 release package.

---

## 11.5 No automatic temporal aggregation

The package provides observations at daily, monthly, and annual resolutions. It does not aggregate or disaggregate user model outputs. Users must compare model output and reference observations at consistent temporal resolutions.

---

## 12. Recommended citation and documentation language

For manuscript, data descriptor, or user documentation, the minimal release product can be described as follows:

> The minimal release package is distributed as a lightweight station-reference matrix dataset derived from the full S8 suspended sediment release. It contains daily, monthly, and annual station-by-time NetCDF matrices for discharge, suspended sediment concentration, and suspended sediment load, together with compact station, source-station, and source-dataset catalogs. The package is designed for direct model-observation comparison and basic provenance tracing through `cluster_uid`, `time`, `resolution`, and `selected_source_station_uid`, while advanced provenance, GIS, climatology, satellite-validation, and overlap-diagnostic products remain in separate full or extension packages.

A shorter version is:

> The `sed_reference_release_minimal` package provides model-ready daily, monthly, and annual station-by-time sediment reference matrices with compact catalogs for station selection and selected source-station provenance tracing.

---

## 13. Summary

The `sed_reference_release_minimal` package is the recommended first-use distribution format for users who need simple, model-ready suspended sediment reference observations.

Its core components are:

```text
sed_reference_timeseries_daily.nc
sed_reference_timeseries_monthly.nc
sed_reference_timeseries_annual.nc
    Daily/monthly/annual station-by-time matrix products for Q, SSC, and SSL.

station_catalog.csv
    Compact cluster/resolution-level station catalog for station selection.

source_station_catalog.csv
    Compact source-station/resolution-level provenance catalog.

source_dataset_catalog.csv
    Compact source-dataset-level metadata catalog.

README.md
release_inventory.csv
minimal_release_validation_report.csv
    Documentation, inventory, and validation files for public distribution.
```

The most important user-facing keys are:

```text
cluster_uid
resolution
time
selected_source_station_uid
source_station_uid
source_name
```

For most model validation workflows, users should begin with:

```text
station_catalog.csv
sed_reference_timeseries_daily.nc
sed_reference_timeseries_monthly.nc
sed_reference_timeseries_annual.nc
```

For basic provenance tracing, users should use:

```text
selected_source_station_uid
source_station_catalog.csv
source_dataset_catalog.csv
```

In summary, `sed_reference_release_minimal` is a lightweight, matrix-first, model-evaluation-ready version of the S8 suspended sediment reference dataset. It is suitable for public user release, quick analysis, and model validation, while the full S8 release remains the authoritative source for complete provenance, overlap diagnostics, GIS sidecars, climatology, and satellite-validation products.
